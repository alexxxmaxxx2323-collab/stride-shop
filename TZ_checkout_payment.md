## 1. Гостевой режим + авторегистрация на чекауте

**Как у Lamoda: единая страница оформления, БЕЗ всплывающих окон.**

- Пользователь без регистрации: смотрит каталог, добавляет в корзину и избранное, выбирает размер/цвет, начинает оформление.
- Текущую стандартную форму регистрации, которая открывается при заказе, **убрать**.
- На странице оформления — блок **«Кто заберёт заказ?»** прямо inline:
  - **Имя\*** · **Фамилия\***
  - **Телефон\*** · **Email\***
- При нажатии «Оформить заказ» аккаунт **создаётся автоматически** по этим данным (отдельная регистрация не нужна).
- Вверху страницы — мягкий баннер-подсказка «Авторизуйтесь, чтобы видеть персональные условия и отслеживать заказы» (не принуждение).

### Согласия (решение: click-wrap, как у Lamoda)
- **Один** чекбокс: ☑ «Получать уведомления о заказах и скидках» — **стоит по умолчанию** (маркетинг).
- Согласие с офертой/обработкой ПДн — **текстом под кнопкой**: «Нажимая "Оформить заказ", вы соглашаетесь с офертой» + кликабельная ссылка на документ (открывает текст). Без обязательной галочки.

## 2. Подтверждения и личный кабинет

- **SMS** — после оформления заказа на указанный телефон шлём СМС для подтверждения заказа (как у Lamoda: «После оформления заказа пришлём СМС для подтверждения»).
- **Email** — после авторегистрации на почту приходит письмо «подтвердите адрес».
- **Личный кабинет** создаётся автоматически после заказа; в нём — вся история заказов и покупок.

## 5. Оплата — ЮKassa (тестовый магазин / sandbox)

**Решение: реальная интеграция ЮKassa в ТЕСТ-режиме («реальная ЮKassa с заглушкой»).**

- На бэке создаём платёж через API ЮKassa → получаем `confirmation_token`.
- На фронте рендерим **официальный виджет ЮKassa** (`yookassa-checkout-widget`): настоящий вид со всеми способами — СБП, банковская карта (МИР/Visa/MC) с реальными полями ввода, кошельки.
- Оплата тестовыми картами; **реальные деньги не списываются** (это и есть «заглушка в конце»).
- Webhook `payment.succeeded` → заказу ставим `payment_status=paid`. На локалке (без публичного webhook) — короткий fallback, помечающий заказ оплаченным.
- Старую заглушку `/payments/mock` и самодельный выбор СБП/МИР/cod — **заменяем** на виджет ЮKassa.

### Нужно от заказчика
1. Зарегистрировать магазин на **yookassa.ru** (бесплатно).
2. Включить **тестовый магазин** → получить `shopId` + секретный ключ (test).
3. Передать их → пропишем в `backend/.env`: `YOOKASSA_SHOP_ID`, `YOOKASSA_SECRET_KEY`.

### Зависимости
- Пакет `yookassa` в `backend/.venv` (поставить с разрешения — см. CLAUDE.md).
